Over the past ten years, recommendation systems have become increasingly popular as more companies strive to offer customized user experiences. Amazon recommends products you may like based on your browse and purchase history, Netflix recommends movies and TV shows based on your past viewing history, and music platforms like Spotify recommend songs you may like based on your listening history. While these recommendation systems can be built based on various different algorithms, they are all trying to achieve the same goal: use the characteristics of the products/movies/music a user is known to like to figure out the products/movies/music the user may like but hasn’t discovered yet.

In today’s lab, we will focus on using song characteristics to determine whether or not a user will like a song. We will use logistic regression to build a model that predicts the probability a user likes a song using the relevant characteristics of the song. The data in this lab is from the Spotify Song Attributes data set in Kaggle. This data set contains song characteristics of 2017 songs and whether or not the person liked the song. This data set contains the song preferences of a single person; therefore, our the scope of our analysis is limited to the person who created the data set.

See “How Does Spotify Know You So Well?” on Medium.com for more information about Spotify’s recommendation algorithms.
